## Горько‑сладкие линии — маленькая визуальная новелла (HTML / CSS / JS)

Это маленькая, полностью автономная визуальная новелла, которая работает целиком в браузере: **без бэкенда** и **без сборщиков**.  
Всё написано на чистых **HTML5 + CSS3 + vanilla JavaScript**, так что игру можно просто открыть и играть.

---

### 1. Структура проекта

- **index.html** – Главная страница и разметка игры
- **style.css** – Внешний вид, расположение элементов, переходы, эффекты кнопок
- **story.js** – Все данные истории (главы, сцены, варианты выбора, текст)
- **script.js** – Игровой движок и логика (состояние, переходы, печать текста, условия разблокировки)
- **assets/**
  - **backgrounds/** – Фоновые изображения для сцен
  - **characters/** – (Необязательно) изображения персонажей, если захотите добавить
  - **audio/** – (Необязательно) музыка и звуки, если захотите добавить

Вы можете переименовывать картинки или добавлять новые, главное — обновлять пути к ним в `story.js`.

---

### 2. Как запустить игру локально

1. **Скачайте / скопируйте** всю папку `visual-novel-game` на свой компьютер.
2. Убедитесь, что в папке есть:
   - `index.html`
   - `style.css`
   - `script.js`
   - `story.js`
   - `assets/` (с нужными вам картинками)
3. Откройте эту папку.
4. **Дважды кликните** по файлу `index.html` (или ПКМ → «Открыть с помощью…» → ваш браузер).
5. Игра должна сразу запуститься в браузере по умолчанию.

> Совет: для лучшего опыта используйте современный десктопный браузер: Chrome, Edge, Firefox или Safari.

---

### 3. Как редактировать историю (понятно новичку)

Весь сюжет находится в файле **`story.js`** в одном объекте JavaScript с именем `story`.

Пример структуры одной сцены:

```javascript
const story = {
  firstSceneId: "1.1",
  scenes: {
    "1.1": {
      chapter: "Глава 1 — Детство",
      background: "assets/backgrounds/childhood_room.jpg",
      text: "На столе лежат две старые фотографии...",
      choices: [
        { text: "Посмотреть на девочку", next: "1.2_girl" },
        { text: "Посмотреть на мальчика", next: "1.2_boy" }
      ]
    }
  }
};
```

- **`"1.1"`** – ID сцены (вы можете придумывать свои, например `"2.1"`, `"final"` и т.д.).
- **`chapter`** – Текст, который показывается в плашке главы в левом верхнем углу.
- **`background`** – Путь к файлу фоновой картинки.
- **`text` / `textVariants`** – Текст сцены (движок сам печатает его «печатной машинкой»).
- **`choices`** – Массив кнопок:
  - `text` – надпись на кнопке;
  - `next` – ID сцены, в которую переходим;
  - дополнительные поля (`requiresVisitedObjectsAtLeast`, `requiresAllDaughterScenes`, `restart`, `correct`, `wrong`) — читаются движком и управляют логикой.

Чтобы изменить сюжет, достаточно редактировать **`story.js`**.  
Трогать `script.js` не нужно, если вы не хотите менять сам движок.

### 4. Как выложить игру на GitHub

Нужно:

- аккаунт **GitHub**;
- установленный **Git** (опционально; можно и через веб).

#### 4.1. Создаём новый репозиторий

1. Зайдите на GitHub и нажмите **“New”** (создать репозиторий).
2. Придумайте имя, например: **`visual-novel-game`**.
3. Выберите **“Public”**.
4. Остальные настройки можно оставить по умолчанию.
5. Нажмите **“Create repository”**.

#### 4.2. Заливаем файлы (два варианта)

##### Вариант A — через Git (командная строка)

1. Откройте терминал и перейдите в папку с проектом:

   ```bash
   cd /path/to/visual-novel-game
   ```

2. Инициализируйте новый git‑репозиторий:

   ```bash
   git init
   ```

3. Добавьте все файлы:

   ```bash
   git add .
   ```

4. Сделайте коммит:

   ```bash
   git commit -m "Initial visual novel game"
   ```

5. Подключите удалённый репозиторий на GitHub (замените `YOUR_USERNAME` и имя репозитория):

   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/visual-novel-game.git
   ```

6. Отправьте код:

   ```bash
   git branch -M main
   git push -u origin main
   ```

##### Вариант B — через веб‑интерфейс GitHub (без Git)

1. На странице репозитория нажмите **“Upload files”**.
2. Перетащите туда всё содержимое папки **`visual-novel-game`**:
   - `index.html`
   - `style.css`
   - `script.js`
   - `story.js`
   - папку `assets/`
3. Пролистайте вниз и нажмите **“Commit changes”**.

---

### 5. Включаем GitHub Pages

GitHub Pages позволяет разместить эту игру как статический сайт.

1. Откройте свой репозиторий на GitHub.
2. Нажмите вкладку **“Settings”** (вверху).
3. В левом меню найдите и нажмите **“Pages”** (иногда это раздел “Code and automation” → “Pages”).
4. В разделе **“Source”** или **“Build and deployment”**:
   - выставьте **Source** → **“Deploy from a branch”**;
   - выберите ветку `main` (или `master`, если у вас так);
   - папку укажите **`/ (root)`** (важно: **не** `/docs` и не другая подпапка).
5. Нажмите **“Save”**.

GitHub соберёт и опубликует сайт.  
Первый запуск может занять пару минут.

---

### 6. Какой будет ссылка GitHub Pages

Когда GitHub Pages включён:

- Игра будет доступна по адресу:

  ```text
  https://YOUR_USERNAME.github.io/REPOSITORY_NAME/
  ```

  Например, если:
  - имя пользователя: `alexdev`
  - имя репозитория: `visual-novel-game`

  То адрес будет:

  ```text
  https://alexdev.github.io/visual-novel-game/
  ```

- Вы (и любой другой человек) сможете открыть эту ссылку и играть прямо в браузере.

### 7. Как кастомизировать игру под себя

- **Поменять картинки**: положите свои `.jpg` / `.png` в `assets/backgrounds/` и обновите пути в `story.js`.
- **Поменять текст**: редактируйте поля `text` (или `textVariants`) у сцен в `story.js`.
- **Добавить сцены**: скопируйте существующую запись сцены, дайте ей новый ID и укажите этот ID в `next: "newSceneId"` у нужной кнопки.

Главное — держать вместе:

- `index.html`
- `style.css`
- `script.js`
- `story.js`

и чтобы в `index.html` скрипты подключались именно в таком порядке:

```html
<script src="story.js"></script>
<script src="script.js"></script>
```

Тогда игра будет работать без сборки и без сервера — просто по открытию `index.html` или через GitHub Pages.
